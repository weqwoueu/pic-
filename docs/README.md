# 文档关系说明（先看这个）

本文不是使用手册，而是说明本仓库几份文档之间的分工关系，避免“该去哪里看什么”混淆。

---

## 文档分工（强约定）

- **根目录 `README.md`：** 项目总览（结构、功能、输出、作用）+ 历史流程保留。  
- **`docs/DEPLOY.md`：** 部署与代码使用主文档（从拉起环境到跑通最小算例、排错）。  
- **`PIC-IFE_GEC/README.md`：** 只在进入工程子目录后看的本地说明与命令摘要。

---

## 建议阅读顺序

| 顺序 | 文档 | 用途 |
|------|------|------|
| 1 | [../README.md](../README.md) | **项目总览**：结构、功能、输出、作用 |
| 2 | [DEPLOY.md](./DEPLOY.md) | **部署 + 使用**：克隆、`module`、最小算例、排错 |
| （可选） | [../PIC-IFE_GEC/README.md](../PIC-IFE_GEC/README.md) | 只进了 **`PIC-IFE_GEC/`** 时：说明本目录即工程根、文档链接、命令摘要 |

**目标：** 任何新同学都能按 **README（看懂项目）→ USAGE（复制命令跑通）** 的顺序完成最小算例验证。
